# Lab 02 - Deleguer les acces sur le projet DocuShare

## Contexte

Les utilisateurs internes existent maintenant.

Il faut leur donner un premier niveau d'acces sur le projet de developpement DocuShare.

## Objectif

Creer le projet :

- `docushare-dev`

Puis attribuer :

- `dev1` = `edit`
- `ops1` = `admin`
- `auditor1` = `view`

Vous devez faire cette delegation depuis la console :

- creation du projet
- ouverture de l'onglet `Project access`
- attribution des roles aux utilisateurs

## Etapes

1. Ouvrez `Home -> Projects`.
2. Cliquez sur `Create Project`.
3. Creez le projet `docushare-dev`.
4. Ouvrez le projet `docushare-dev`.
5. Ouvrez l'onglet `Project access`.
6. Cliquez sur `Add access`.
7. Creez successivement trois attributions d'acces.

Important :

- les roles `admin`, `edit` et `view` sont des `ClusterRole` par defaut d'OpenShift ;
- ici, on les attache localement au projet `docushare-dev` ;
- selon l'ecran de console utilise, le formulaire `RoleBindings` peut ne pas afficher clairement `edit` et `view` dans la liste.

Donc pour ce lab, privilegiez :

- `Project access -> Add access`

Si la liste des roles n'affiche pas `edit` ou `view`, utilisez le mode YAML dans la console avec les exemples plus bas.

Pour chaque utilisateur, remplissez comme ceci :

## 1. Donner accès edit à dev1

```text
Nom : dev1-edit
Espace de noms : docushare-dev
Nom du rôle : edit
Objet : Utilisateur
Nom de l’objet : dev1
```

```text
Nom de l’objet : auditor1
```

Puis **Create**

## Ce que ça signifie

* `dev1` : peut déployer, modifier app, PVC, services
* `ops1` : admin du projet
* `auditor1` : lecture seule

---

## Validation attendue

Dans `RoleBindings`, retrouver :

```text
dev1-edit
ops1-admin
auditor1-view
```

## Validation attendue

Dans `Project access`, vous devez voir :

- `dev1` avec `edit`
- `ops1` avec `admin`
- `auditor1` avec `view`

## Ce qu'il faut retenir

- la delegation namespaced permet de ne pas donner `cluster-admin` a tout le monde ;
- `edit`, `admin` et `view` n'ont pas le meme niveau de pouvoir ;
- un projet bien gere commence par des acces clairs et explicites.
